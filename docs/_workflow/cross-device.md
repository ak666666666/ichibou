# どの端末からでも「続きから」作業するための規約

> ゴール: PC / Web Claude Code / スマホ / その他、**どの端末で作業を中断しても、別端末でそのまま続ける」状態を作る。
>
> 原則: **GitHub リポジトリを唯一の正本にする**。ローカルの記憶やチャット履歴を信用しない。

## 1. 保存場所のルール

すべての「思考ログ」「設計メモ」「仕分け表」は、該当プロジェクトのリポジトリに入れる。

```
<repo>/
  docs/
    _workflow/                # このドキュメントや共通規約
      cross-device.md
    <project-slug>/           # 1 テーマ = 1 フォルダ
      README.md               # 概要 / 使い方
      STATE.md                # 現状と次の一手（唯一の正本）
      log-template.md         # 思考ログのひな形
      log-01.md, log-02.md... # ログは時系列で番号を振る
```

プロジェクト名はケバブで表し、そのままフォルダ名とブランチ名に使う（例: `memo-debug`, `voice-recorder-ios`, `kpi-dashboard`）。

## 2. ブランチと PR

- ブランチ名: `claude/<目的>-<短いID>` で固定。例: `claude/organize-memo-features-V6Iyg`
- 作業中は**常に draft PR を開きっ渡しにしておく**。
  - どの端末からでも PR URL を見れば「今どこまで進んだか」がわかる
  - コメントで議論スレッドもそこに集約される
- マージしたらブランチを削除し、次のテーマに新しいブランチを切る。

## 3. 端末を切り替える手順

### 作業を中断する側

1. その時点の `STATE.md` を書き換える（何が済んで、次に何をやるか）
2. 進行中の思考も `log-NN.md` に吹き出しておく
3. **commit & push**。チャットを閉じて OK

### 作業を再開する側

1. そのブランチを `git fetch origin && git switch <branch>` （初回は `git switch -c <branch> --track origin/<branch>`）
2. `STATE.md` を読む → 「次の一手」から始める
3. どうしてそうしたかを思い出す必要があれば、`log-NN.md` を逆順に読む

### Web Claude Code 側で動くとき

- この環境は GitHub MCP だけがストレージ。**ローカルに保存したチャットログは見えない**。
- したがって PC 側で思考ログを markdown 化→commit→push しておくのが必須。

## 4. STATE.md の書き方（最重要）

`STATE.md` はこの仕組みの要。次のセクションを必ず含める:

- **仕分け表** — 課題を ✅ 実装済み / ⏳ 未実装 / ❌ 不要 / ❔ 要確認 に分けて一覧
- **次の一手** — 今このブランチで何をやるべきか、チェックボックス 3〜5 件
- **ブロッカー** — 詰まっている項目があればその原因

「実装を 1 歩進めたら STATE.md も 1 行更新」を徹底する。

## 5. 新しいプロジェクトを始めるとき

1. スラッグを決める（例: `kpi-dashboard`）
2. ブランチを切る: `claude/<目的>-<ID>`
3. このテンプレをコピー:
   - `docs/<slug>/README.md`、`STATE.md`、`log-template.md`、`log-01.md`
4. draft PR を開く
5. スマホや PC で思考ログがたまったら `log-NN.md` に追加して push

## 6. アンチパターン（やってはいけないこと）

- チャット UI のスクロールを「それがログ」と見なす — そのセッションを閉じたら消える
- ローカルの `~/Documents/メモ.txt` にそのまま保存 — Web Claude Code から見えない
- 1 件の PR に複数テーマを詰め込む — 「進捗 = PR と STATE.md」の対応が崩れる

## 7. チェックリスト（1 作業セッションごと）

- [ ] 作業前に `git pull` したか
- [ ] 作業中の思考を `log-NN.md` に反映したか
- [ ] `STATE.md` の「次の一手」を更新したか
- [ ] commit & push したか
- [ ] draft PR を見て CI / コメントに議論もれがないか
