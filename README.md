# 一望(ichibou)— 公開・インストール手順

このフォルダは、一望アプリの完全なファイル一式です。
これをGitHub Pagesにアップロードすれば、スマホ・Fold・PCから同じURLでアクセスでき、
Galaxy Z Fold 7のホーム画面にアイコンとして追加できます(PWA化)。

---

## 含まれているファイル

- `index.html` — アプリ本体
- `manifest.json` — PWAの設定(アプリ名・アイコン・起動モード)
- `sw.js` — Service Worker(PWA化に必要な最小構成)
- `icon.svg` — ブラウザタブ用アイコン
- `icon-192.png` — ホーム画面用アイコン(小)
- `icon-512.png` — ホーム画面用アイコン(大)
- `icon-maskable.png` — Androidの丸型アイコン用
- `README.md` — この手順書

## GitHub Pagesに公開する手順(約10分)

### ステップ1: GitHubアカウントを作る(持ってなければ)
1. https://github.com/ にアクセス
2. 「Sign up」から無料アカウント作成
3. メール認証を済ませる

### ステップ2: 新しいリポジトリを作る
1. GitHubにログイン後、右上の「+」→「New repository」
2. Repository name に `ichibou` と入力(好きな名前でOK)
3. 「Public」を選択(無料でGitHub Pagesを使うにはPublicが必要)
4. 「Create repository」をクリック

### ステップ3: ファイルをアップロード
1. 作成したリポジトリのページで「uploading an existing file」リンクをクリック
2. このフォルダの中身(index.html, manifest.json, sw.js, icon系3つ, icon.svg)を全部ドラッグ&ドロップ
3. 下の「Commit changes」をクリック

### ステップ4: GitHub Pagesを有効化
1. リポジトリのページで「Settings」タブをクリック
2. 左メニューの「Pages」をクリック
3. Source の下で:
   - Branch: `main` を選択
   - Folder: `/ (root)` のまま
4. 「Save」をクリック
5. 1〜2分待つと、上部に「Your site is live at https://あなたのユーザー名.github.io/ichibou/」と表示される

このURLが、あなたの一望のURLです。

## Fold / スマホのホーム画面に追加する(PWA化)

### Android (Galaxy Z Fold 7など)の場合
1. Chromeで公開したURLを開く
2. アドレスバー右の「︙」メニューをタップ
3. 「アプリをインストール」または「ホーム画面に追加」をタップ
4. 「インストール」をタップ
5. ホーム画面に「一望」アイコンが追加される
6. タップすると、ブラウザUIなしでアプリのように起動する

### iPhoneの場合
1. Safariで公開したURLを開く
2. 下部の「共有」ボタン(□と↑のアイコン)をタップ
3. 「ホーム画面に追加」をタップ
4. 「追加」をタップ

## 他の人に見せる

公開したURL(例: `https://あなたのユーザー名.github.io/ichibou/`)を伝えるだけ。
相手もブラウザで開けます。ただし、データはそれぞれのブラウザの中にだけ保存されるので、
あなたが書いたもややプロジェクトは相手には見えません(プライバシー的には安全)。

## 更新する時

index.htmlの中身を書き換えたい時:
1. GitHubのリポジトリで `index.html` をクリック
2. 右上の鉛筆アイコン(Edit)をクリック
3. 編集して「Commit changes」
4. 1〜2分でサイトに反映される

## データのバックアップ(重要)

現時点のバージョンでは、データは **ブラウザのlocalStorageに保存** されます。
つまり:
- ブラウザの履歴を消すとデータも消える
- 別のデバイス(PCとスマホ)では別々のデータになる
- PC⇔スマホの自動同期はない

**週に1回、週次タブから「プロンプトをコピー」ボタンで全データをテキストとしてコピー**
できるので、それをGoogleドキュメント等に貼っておくと、バックアップになります。

本格的なクラウド同期(Supabase等)は、1週間使って本当に続きそうだと判断した後で
実装する方が無駄がありません。

## 独自ドメインにしたくなったら(後日の選択肢)

`ichibou.jp` や `lbc-ichibou.com` のような独自ドメインにしたくなったら:
1. お名前.comやCloudflare等でドメインを購入(年1000〜3000円)
2. GitHubリポジトリの Settings → Pages → Custom domain で設定
3. ドメイン側のDNSレコードをGitHubに向ける

これは後でも可能です。まずは `.github.io` のURLで1ヶ月使って、
本当に育てる価値があると分かってから判断するのが合理的です。

## やらない機能(意図的に実装しないリスト)

一望は「脳の外部化」が目的のため、機能を増やしすぎると逆に脳の圧迫になる。
以下は検討した結果、**今は実装しない**と明示的に決めた機能。再検討するときの出発点になる。

### Tier 3 — 保留(再検討の余地あり)

- **プロジェクト優先度機能**:メンタルモデルが未確立な状態で並び替えUIを足すと、
  並び替えそのものに脳メモリを使う罠になる。優先度の付け方が言語化できてから検討。
- **画像/動画/ファイル添付**:外部脳の役割とファイルストレージの役割は別物。
  localStorage 容量と Supabase 課金が即発生する。Cowork / Notion / Claude 側に置く方が筋が良い。
- **カレンダー・通知機能**:OSの通知/Googleカレンダーで足りる可能性が高い。
  「通知で割り込んでくる」のは脳外部化思想と逆方向。
- **プロジェクト進捗ログ欄(前回どこまで)**:思考ログが既に進捗ログを兼ねている。
  二重管理になるため、まずは思考ログを「思考+進捗ログ」と再定義する方を優先。
- **自動タグ・フォルダ振り分け / ジャンル別ギャラクシーノート的UI**:
  自動分類は脳の容量を逆に食う。手動の3層モデル(プロジェクト/もや/雑メモ)で十分。

### Tier 4 — 一望の範疇外(別プロジェクト化)

- ゴルフ・バトミントン上達記録・マインドマップ別タブ:別アプリ
- Claude Code連携で「実装完了を自動でチェック」:別ツール(GitHub webhook + Claude Code)
- Cowork→一望の自動同期高度化:既に `/sync-ichibou` で実現済み、超過機能化の罠

### 判断の物差し

新しい機能を足したくなったら、次の問いを通す:
1. これは**脳の外部化**を進めるか、それとも**新しい脳の負荷**を生むか?
2. 既存の3層(プロジェクト / もや / 雑メモ)で表現できないか?
3. 一望ではなく、Cowork / Claude / Notion / OS で済まないか?

3つとも「YES, 一望に必要」と即答できないなら、このリストに追加して保留する。

## トラブルシューティング

**Q: ホーム画面追加ボタンが出てこない**
→ HTTPS接続になっているか確認(GitHub Pagesは自動でHTTPSになります)。
  Chromeを最新版にアップデート。

**Q: アイコンが漢字じゃなくて変な表示になる**
→ icon-192.png と icon-512.png が正しくアップロードされているか確認。

**Q: 公開URLでアクセスしたけど真っ白**
→ 公開まで1〜2分かかる場合があります。少し待ってからリロード。
  それでもダメなら、Settings → Pages の設定を再確認。
